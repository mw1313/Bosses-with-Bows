# Page CRO

## Before Analyzing — Identify

1. **Page type:** Homepage / landing page / pricing / feature / blog
2. **Primary conversion goal:** Purchase / sign up / inquiry / notify me
3. **Traffic source:** Organic / paid / social / email (message-match matters)

## 7 CRO Dimensions — in Order of Impact

**1. Value proposition clarity (highest impact)**
- Can a visitor understand what this is and why they should care in 5 seconds?
- Benefit-focused, not feature-focused
- Customer's language, not company jargon
- One clear most important thing — not everything at once

**2. Headline effectiveness**
- Communicates core value prop?
- Specific enough to be meaningful?
- Matches traffic source messaging?

**3. CTA placement, copy, hierarchy**
- One clear primary action visible without scrolling
- Button copy = value, not just command ("Start building" not "Submit")
- CTAs repeated at key decision points
- Clear primary vs. secondary hierarchy

**4. Visual hierarchy and scannability**
- Scanning reveals the main message
- Most important = most prominent
- White space used intentionally
- Images support (not distract from) message

**5. Trust signals and social proof**
- Customer logos, attributed testimonials with specifics, case study numbers
- Placed near CTAs and after benefit claims
- Review scores, security badges where relevant

**6. Objection handling**
- Price/value · "Will this work for me?" · Implementation difficulty · "What if it doesn't work?"
- Addressed via FAQ, guarantees, process transparency, fit/not-fit section

**7. Friction points**
- Form fields (fewer is more)
- Unclear next steps
- Navigation pulling focus from CTA
- Mobile experience
- Load time

## Output Format — Always This Structure

**Quick Wins (Implement Now)** — easy changes, immediate impact
**High-Impact Changes (Prioritize)** — bigger effort, significant conversion lift
**Test Ideas** — hypotheses worth A/B testing, not assuming
**Copy Alternatives** — 2–3 variants per key element with rationale

Never give a flat list. Every recommendation = rationale tied to which CRO dimension it addresses. Never recommend A/B testing as a substitute for obvious fixes.

## Page-Specific Frameworks

**Homepage:** Handle cold visitors. Clear positioning. Fast path to conversion. Serve both "ready to buy" and "still researching."

**Landing page:** Single CTA. Remove nav if possible. Message-match with traffic source. Complete argument on one page.

**Pricing page:** Clear plan comparison. Recommend one plan. Address "which plan is right for me?" anxiety.

**Feature page:** Feature → benefit → use case → clear path to try/buy.

**Blog post:** Contextual inline CTAs matching the content topic.
